计算机系统结构期末复习整理

题型

四种题型：

  • 单选
  • 简答
  • 计算
  • 综合应用

第一章

  • 六个层次

    • 物理机（硬件/固件实现）

      • 应用语言机器
      • 高级语言机器
      • 汇编语言机器
      • 操作系统机器

    • 虚拟机（软件实现）

      • 传机器语言机器
      • 微程序机器

  • 系统结构

    • 传统机器程序员所看到的计算机属性（机器语言级）
    • 传统定义：实质是确定计算机系统中软硬件的交界面，界面之上是软件实现的功能，界面之下是硬件和固件实现的功能

  • 透明性

    • 本来存在的事物或属性，但从某种角度看又好像不存在的概念称为——透明性

  • 组实构区别与联系

    • 系统结构定义在前已述
    • 计算机组成：计算机系统结构的逻辑实现，包含物理机器级中的数据流和控制流的组成以及逻辑设计，着眼于物理机器级内各事件的排序方式与控制方式、各部件的功能以及各部件之间的联系。
    • 计算机实现：指的是计算机组成的物理实现，包括处理机、主存等部件的物理结构等，着眼于器件技术和微组装技术，其中器件技术在实现技术中起主导作用。

  • Flynn分类法

    • 单指令流单数据流——SISD——传统的顺序处理机器
    • SIMD——阵列处理剂
    • MISD——没有实际的机器
    • MIMD——多处理机机器

  • 分类标准——指令流和数据流的多倍性

  • Amdahl定律（Exp1.2、作业习题）

    • $T_n=T_0(1-F_e+\frac{F_e}{S_e})$
    • $S_n=\frac{T_0}{T_n}=\frac 1 {1-F_e-\frac{F_e}{S_e}}$

  • CPI公式

    • $CPU时间(s)=执行程序所需的时钟周期数×时钟周期时间$
    • $CPI(clock/instuction)=执行程序所需的时钟周期数/所执行的指令条数$
    • $CPU时间=IC×CPI×时钟周期时间=\sum^n_{i=1}(CPI_i × \frac{IC_i}{IC})$
    • $MIPS(instuction/s)=\frac{所执行的指令条数}{CPU时间×10^6}=\frac{时钟频率}{CPI×10^6}$

  • 局部性

    • 时间局部性——程序即将用到的信息很可能就是目前正在使用的信息
    • 空间局部性——程序即将用到的信息很可能与目前正在使用的信息在空间上相邻或者临近

  • 三种设计方法

    • “由上往下”设计
    • “由下往上”设计
    • “从中间开始”设计
      • “中间”指的是层次结构中的软硬件的交界面，目前一般是在传统机器语言机器级与操作系统机器级之间

  • 软件移植

    • 定义
      • 指一个软件可以不经修改或者少量修改就可以由一台计算机移植到另一台计算机上运行。
    • 三种方法
      • 统一高级语言
      • 采用系列机
      • 模拟与仿真

  • 软件兼容

    • 向后（前）——某个时期投入市场的某种型号计算机编制的程序，不加修改就能运行在它之后（前）投入市场的计算机
    • 向上（下）——按某档计算机编制的程序，不加修改就能运行于比它高（低）档的计算机
    • 向后兼容是肯定要做到的，它是系列机的根本特征

  • 模拟和仿真的区别

    • 主要区别在于解释执行所用的语言
    • 仿真是用微程序解释执行的
    • 模拟是用机器语言解释执行的

  • 并行性

    • 同时性
      • 字串位串
      • 字串位并
      • 字并位串
      • 全并行
    • 并发性

  • 什么时候具有初步并行性？——字串位并

  • 什么时候正式进入到并行处理？——字并位串

  • 提高并行性的技术途径

    • 时间重叠（引入时间因素）——并发性
    • 资源重复（引入空间因素）——同时性
    • 资源共享

  • 习题1.9、习题1.11

第二章

  • Huffanman思想描述——当各种事件发生的概率不均等时，可以对发生概率最高的事件用最短的位数（时间）来表示（处理），而对于出现概率较低的事件，则可以用较长的位数（时间）来表示（处理），从而使平均位数（时间）缩短。
  • 如何构造HuffmanTree？（会使用，无需背诵）
  • 信息熵$$H=-\sum_{i=1}^np_ilog_2p_i$$——表示用二进制编码表示n个码点时，理论上的最短平均编码长度。
    • $log_22=1,log_23=1.6,log_25=2.3,log_27=2.8$
  • 平均码长 $L=\sum_{i=1}^np_il_i$
  • 信息冗余量$R=\frac {L-H} L$
  • 扩展编码 n1/n2/n3
    • 前m位中使用n1个码点作为操作码
    • 剩余的$2^m-n_1$个码点用作扩展标志，可以和后面n2/n3能够表示的操作码总数相乘得到剩余能够表示的操作码
    • 将n1+扩展码数得到总操作码数
  • CISC和RISC
    • CISC复杂指令集计算机——增强指令功能，把越来越多的功能交由硬件来实现，指令的数量也越来越多
    • RISC精简指令集计算机——尽可能地把指令系统简化，不仅指令的条数少，而且指令的功能也比较简单

第三章

  • 分类

    • 等级

      • 部件级流水线
      • 处理机流水线
      • 系统级流水线

    • 功能

      • 单功能流水线
      • 多功能流水线

    • 多功能流水线进一步分为

      • 静态流水线
      • 动态流水线

    • 是否存在反馈回路

      • 线性流水线
      • 非线性流水线

    • 任务流入和流出的顺序是否相同

      • 顺序流水线
      • 乱序流水线

  • 流水线的性能指标

    • 吞吐率

      • 单位时间内流水线所完成的任务数量或输出结果的数量

      • $TP=\frac n {T_k}$

      • 各段时间均相等的流水线

        • $TP = \frac{n}{(n-1+k)\Delta t}$

      • 各段时间不相等的流水线

        • $TP=\frac{n}{\sum\Delta t_i+(n-1)\Delta t_{max}}$

    • 加速比

      • 顺序处理方式处理一批任务所用的时间与流水线使用流水处理方式处理同一批任务所用的任务

      • $S=\frac{T_s}{T_k}$

      • 各段时间均相等

        • $S=\frac{nk}{n-1+k}$

      • 各段时间不相等的流水线

        • $S=\frac{n\sum\Delta t}{\sum \Delta t + (n-1)\Delta t_{max}}$

    • 效率

      • 流水线中的设备实际使用的时间与整个运行时间的比值

      • 各段时间相等

        • $E=\frac n {n-1+k}={TP}* {\Delta t}$

      • 各段时间不相等

        • $E=\frac{n*\sum\Delta t}{k(\sum \Delta t + (n-1)\Delta t_{max})}$

    • 瓶颈段的消除

      • 细分瓶颈段——把流水线中的瓶颈段切分为几个独立的功能段，从而使流水线各段的处理时间都相等。
      • 重复设置瓶颈段

  • 单流水线非线性的最优调度（不要求背诵，但要求计算应用）

    • 启动距离——向一条非线性流水线的输入端连续输入两个任务之间的时间间隔

    • 禁用启动距离——会引起非线性流水线功能段使用冲突的启动距离

    • 根据预约表写出禁止表

    • 根据禁止表F写出初始冲突向量$C_0$

    • 根据初始冲突向量$C_0$画出状态转换图

      • 假设第二个任务是在与第一个任务间隔j时钟周期流入的，由于第一个任务已经在流水线中前进了j个时钟周期，其相应的禁止表中各元素的值都应该减去j，并丢弃小于等于0的值
      • 对于冲突向量来说，就是逻辑右移j位（左边补零）
      • 新的冲突向量$SHR^{(j)}(C_k)\or C_0$
      • j是允许的时间间隔

    • 根据状态转换图写出最优调度方案

    • 此时求实际吞吐率时的总时间等于{单个任务完成的时间加剩余任务的启动距离}

  • 条件转移和中断

    • 预测分支失败
    • 预测分支成功
    • 延迟分支

第四章

  • 向量处理方式

    • 横向处理方式
    • 纵向处理方式
    • 纵横处理方式
      • 组内纵向
      • 组间横向

    • 数据处理部件处理时间+数据存储部件数据上下线时间（各一拍）+剩余流出时间

第五章

  • 超标量（性能最好）——在一个时钟周期内能够同时发射多条指令的处理机
  • 超流水线——在一个时钟周期内能够分时发射多条指令的处理机
  • 超长指令字——由编译程序在编译时找出指令间潜在的并行性，进行适当调整安排，把多个能并行执行的操作组合在一起，构成一条具有多个操作段的超长指令

第七章

  • 多级

    • 访问时间：$T_1<T_2<…<T_n$
    • 容量:$S_1<S_2<…<S_n$
    • 平均每位价格:$C_1>C_2>…>C_n$

  • 命中率H

    • 定义为——CPU访问该存储系统时，在$M_1$中找到所需信息的概率
    • $H=\frac {N_1} {N_1+N_2}$
    • $N_1$和$N_2$分别访问$M_1$和$M_2$的次数

  • 三级系统

    • Cache、主存储器、磁盘存储器（辅存）
    • Cache-主存 层次——完全由硬件实现，对应用程序员和系统程序员透明
    • 主存-辅存 层次——对应用程序员透明，常常用来实现虚拟存储器

  • 主存-辅存

    • 段式
    • 页式
    • 段页式

  • 二维数组错位存放避免冲突（会计算）

    • 假设单体存储器有m个空间
    • n×n的二维数组
    • 要求$m\ge n$
    • p则取满足$m =2^{2p}$的任意自然数
    • k是数组的第一个元素$a_{00}$所在存储体的体号
    • 体号：$(2^p*i+j+k)mod (m)$
    • 体内地址：i

  • 多体交叉存储器

    • 高位交叉编址——按列（单体存储器）优先编址
    • 低位交叉编址——按行（同体同一对应位置）优先编址

  • 页面置换算法——RAND、FIFO、LRU、OPT（会使用堆栈型算法计算命中率）

  • Cache映像规则（3种）

    • 组相联{组间直接映像、组内全相联}（会设计应用）
    • 全相联映像（冲突最低）
    • 直接映像（冲突最高）

  • 写策略

    • 写直达法
    • 写回法

  • 当发生写不命中时，是否调入相应的块

    • 按写分配法
    • 不按写分配法

  • 习题3.7、习题3.13

    • 中括号、循环
